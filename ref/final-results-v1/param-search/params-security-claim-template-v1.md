# M6 安全表述模板与术语对齐 v1

## 1. 适用范围
- 本模板用于参数重搜阶段（M3-M6）的论文与发布文案。
- 目标是避免“工程可运行/筛选达标”被误写成“严格证明已完成”。
- 适配对象：`simple` 路径、`targeted classical` 口径、`proxy-v1` 自动筛选模型。

## 2. 术语对齐（与主仓文档）
| 术语 | 本项目定义 | 对齐文档 |
|---|---|---|
| 目标筛选达标（targeted filter pass） | 在 `proxy-v1` 下，`claimed_security_bits >= target_bits` 的自动化筛选结果。 | `参数重搜里程碑.md`、`thesis-notes-stark-v2.md` |
| 严格证明（formal proof） | 对完整构造在明确模型下给出可审计、可复核的形式化安全证明。 | `thesis-notes-stark-v2.md`（边界声明） |
| 工程证据（engineering evidence） | 基准测试、篡改检出、跨后端一致性、回归矩阵等可复现实验结果。 | `release-checklist-v2.md` |
| 发布口径（release wording） | 对外说明中明确区分“筛选达标/工程验证”与“严格证明”。 | `release-checklist-v2.md` |

## 3. 允许使用的结论句式（可直接复用）
### 3.1 论文正文（参数筛选段）
- “在本工作中，我们采用 `proxy-v1` 规则对候选参数进行自动筛选；结论仅表示在目标场景下的**筛选达标**，不等价于完整构造的严格证明安全结论。”
- “预算退化分析显示，当签名预算由 `q` 提升至更高量级时，组合层下界将按 `log2` 罚分下降，因此需在给定预算前提下解释安全主张。”

### 3.2 论文结论段（保守口径）
- “本文给出的推荐参数组满足既定工程目标与筛选口径，并在性能/约束指标上形成可复现实证；严格证明仍属于后续工作。”

### 3.3 发布说明（Release Note）
- “当前版本提供参数筛选与实验验证结果（性能、约束、一致性），对应工程证据层面的可复现结论；不宣称已完成完整形式化安全证明。”

## 4. 禁用句式（避免过度主张）
- “该参数组已被证明达到 128-bit 安全。”（禁用，除非已有完整证明材料）
- “本实现与标准参数在全部安全性质上等价。”（禁用，当前证据不足）
- “预算变化不影响安全结论。”（禁用，与 M6 退化分析冲突）

## 5. 报告结构建议（参数重搜章节）
- 小节 A：模型与口径（`simple`、`targeted classical`、`proxy-v1`）。
- 小节 B：参数筛选与推荐（M3-M5 结果）。
- 小节 C：预算退化（M6，多预算点表格）。
- 小节 D：边界与未来工作（严格证明、匿名性/可链接性形式化）。
