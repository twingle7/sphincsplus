# Fischlin-SPHINCS+-Poseidon2-STARK 下一步计划 v2

## 文档目的
- 在 v1 已完成的工程基线上，给出“最终目标版”执行规范。
- 最终目标：实现 **SPHINCS+ 在 Fischlin 框架下的盲签展示**，并完成 **完整 STARK 证明**（非骨架、非占位）。
- 统一协议语义、对象边界、证明语句、工程里程碑与验收标准，避免“实现打通但语义不等价”。

## 2. v2 最终语义定义

### 2.1 目标语句（Fischlin 展示口径）
存在私有 witness `(m, r, sigma_com, trace, aux)`，使得：
1. `com = Commit(m || r)`；
2. `Verify(pk, com, sigma_com) = 1`；
3. `trace` 来自一致执行 `spx_p2_trace_verify_com(trace, pk, com, sigma_com)`；
4. `trace` 与 `witness rows`、AIR 约束、proof transcript 一致；
5. 对外验证者仅见 `(com, pi_F, public_ctx)`，不见 `sigma_com`；
6. 展示验证结论等价于“存在一个有效签名 witness”。

### 2.2 Fischlin 盲签约束（v2 口径）
- Issuer 不应获得可链接到最终展示对象的 `m/r` 细节。
- Holder 侧展示对象满足“可验证、不可伪造、最小泄露”。
- `public_ctx` 仅承载公开绑定上下文，不承载私密 witness 数据。
- 盲签语义与 STARK 语义同时成立：协议层不可链接 + 证明层可验证。

### 2.3 可见性边界（最终版）
- 公开输入：`pk, com, public_ctx`。
- 私有 witness：`m, r, sigma_com, trace, aux`。
- 对外对象：`Show_v2 = (com, pi_F_v2, public_ctx)`。
- 强约束：`pi_F_v2` 不得序列化 `sigma_com` 明文。

## 3. 架构冻结（v2）

### 3.1 分层职责
- `show/*`：展示层 API，仅处理 `Show_v2` 的生成与验证。
- `stark/*`：witness 构造、AIR 约束、prover/verifier、proof 编解码。
- `ffi_v1`：保持 ABI 稳定；通过 proof version 和 feature flag 兼容升级。
- `ref` 验签 helper：仅用于 prover 内部一致执行与重放，不对外暴露签名本体。

### 3.2 版本策略
- 保留 `pi_F_v1` 解析以兼容历史样本。
- 新增 `pi_F_v2` 作为默认输出格式。
- 迁移期允许“验证兼容双版本、生成仅输出 v2”。
- `reserved` 字段不再承载跨语义大扩展；跨语义升级必须升版本号。

### 3.3 一致性原则
- C 与 Rust 后端必须共享同一 proof statement（同一公开输入语义）。
- 任一后端接受/拒绝结果必须与 `verify_full` 语义一致。
- 所有篡改检测都必须落到可复现测试，不允许仅靠文档声明。

## 6. 红线与退出条件
- 红线 1：`pi_F_v2` 中出现 `sigma_com` 明文，直接视为不达标。
- 红线 2：Rust 与 C 在同一输入上出现接受/拒绝不一致，禁止发布。
- 红线 3：盲签流程中出现可链接标识泄露，禁止进入论文结论阶段。

退出条件（可宣告完成）：
- 完成 M12~M17 全部验收；
- 默认路径为 `Show_v2 + pi_F_v2`；
- 端到端实现满足“Fischlin 盲签 + 完整 STARK 证明”双目标。
