# Fischlin 全框架升级路线图 v1（目标语句冻结）

## 1. 文档目的
- 将当前 `Fischlin-like` 展示系统升级为“满足 Fischlin 框架”的最终实现。
- 冻结最终证明语句，避免后续实现与论文语义漂移。
- 定义可执行里程碑、交付物、验收标准与重跑策略（约束/基准/参数搜索）。

## 2. 当前状态（基线）
- 当前 final 路径展示对象为 `Show_v2 = (com, pi_F_v2, public_ctx)`。
- 当前证明主要绑定 `pk/com/public_ctx` 派生语句与 STARK 证明对象一致性。
- 当前 `sigma_com` 合法性主要由 prove 前 guard 检查，尚未完全内生为最终验证语句的一部分。

结论：当前可定义为 `Fischlin-like`，但未达到严格 `Sigma = (C, pi)` 的 Fischlin 语义闭环。

## 3. 最终目标语句冻结（Fischlin-Strict）

### 3.1 最终公开对象与输入
- 展示签名对象：`Sigma = (C, pi)`。
- 验证公开输入：`(pk_sig, pk_E, m_pub, ctx_pub)`（其中 `m_pub` 可按系统策略为明文或其公开摘要）。

### 3.2 最终见证
- 见证：`w = (r, omega2, c, sigma')`。

### 3.3 冻结证明语句（必须同时成立）
- `c = Com(m_pub; r)`。
- `Verify(pk_sig, c, sigma') = 1`。
- `C = Enc(pk_E, c || sigma'; omega2)`。

### 3.4 NIZK 判定语义
- 验证器接受当且仅当存在见证 `w` 使上述三个关系同时成立。
- `pi` 的验证不得依赖“证明生成前外部 guard 成功”作为等价替代。

## 4. 非目标（本次冻结明确排除）
- 不在本阶段重构 SPHINCS+ 底层签名算法本体。
- 不在本阶段追求最优 proof size/latency，只要求功能闭环与安全语义正确。
- 不要求一次性完成所有性能优化（可在收口后做 M+ 优化）。

## 5. 里程碑与交付物

## M18：语句落地与接口改版（Schema Freeze）
- 目标：完成 `Sigma=(C,pi)` 结构切换与接口冻结。
- 代码交付物：
  - 新增/改造展示对象结构体，移除 `Show_v2=(com,pi_f,public_ctx)` 的 final 依赖。
  - 新增 `C` 字段与序列化格式版本（建议 `pi_F_v3` 或等价 final 格式升级）。
  - 明确 `public input` 与 `witness` 边界，写入代码注释与头文件注释。
- 文档交付物：
  - `pi_F` 新格式规范草案（字段、长度、兼容策略、拒绝策略）。
  - API 迁移文档（final/compat 路由与弃用计划）。
- 测试交付物：
  - 格式解析正例/反例测试。
  - 兼容路径隔离测试（legacy 仅走 compat）。
- 验收标准：
  - 新对象路径可编译可运行。
  - final 路径不再以旧展示对象语义作为验证依据。

## M19：关系约束内生化（Com/Verify/Enc 三关系）
- 目标：将三条 Fischlin 关系全部纳入证明语句（不依赖外部 guard 替代）。
- 代码交付物：
  - 承诺关系约束：`c = Com(m_pub; r)`。
  - 签名验证关系约束：`Verify(pk_sig, c, sigma') = 1`。
  - 加密关系约束：`C = Enc(pk_E, c||sigma';omega2)`。
  - 统一语句派生函数，禁止多口径计算。
- 文档交付物：
  - 约束映射表（数学语句 -> AIR/电路列 -> 代码函数）。
  - 安全模型更新（完备性、可靠性、零知识）。
- 测试交付物：
  - 篡改 `r` / `omega2` / `sigma'` / `c` / `C` 的拒绝测试矩阵。
  - 缺失任一关系时必须拒绝的负例集。
- 验收标准：
  - 验证端仅基于 `(C, pi, public inputs)` 判定，不依赖 prove 前检查结论。

## M20：端到端 Fischlin 协议闭环（Issue/Unblind/Show）
- 目标：将盲签发放、去盲化、展示验证形成完整工程闭环。
- 代码交付物：
  - `BlindIssue-Request/Sign/Unblind/ShowProve/ShowVerify` 全链路串联。
  - 角色边界实现：Issuer 不接触展示见证，Verifier 不接触 `sigma'` 明文。
- 文档交付物：
  - 端到端协议时序图与失败路径图。
  - 实现-论文术语映射（变量命名与对象对应关系）。
- 测试交付物：
  - e2e 正常流测试。
  - 重放、篡改、跨会话链接尝试的负例测试。
- 验收标准：
  - e2e 在 final 路径稳定通过，负例拒绝率符合预期。

## M21：数据重跑与参数收口（Benchmark & Search）
- 目标：在新语句下完成可复现数据集，并给出最终参数选择。
- 代码交付物：
  - benchmark 与参数搜索脚本适配新语句/新格式。
  - 结果汇总脚本（表格 + 图 + 失败样本归档）。
- 数据交付物（必须重跑）：
  - 约束规模：columns/rows/constraint count/degree。
  - 性能：prove time / verify time / proof size / memory。
  - 参数搜索：主搜索 + finalists + 回归复测。
- 文档交付物：
  - 参数声明模板（安全位、性能预算、对照组）。
  - 与旧版数据对比报告（仅作历史基线，不作最终结论）。
- 验收标准：
  - 新语句下形成可复现的完整实验包，支持论文与答辩引用。

## M22：发布冻结（Release Freeze）
- 目标：形成可发布、可答辩、可复现的最终包。
- 交付物：
  - release checklist（代码、测试、文档、数据、脚本、样本）。
  - 最终“语句冻结声明”与“兼容策略声明”。
  - one-command 回归入口与环境说明。
- 验收标准：
  - 在干净环境一次性复现核心结果。
  - final 默认路径仅接受 Fischlin-Strict 语句对象。

## 6. 重跑策略冻结（针对历史数据）
- 需要全部重跑：
  - 与证明语句相关的约束统计、证明大小、证明/验证时间、参数搜索结果。
- 可作为历史基线保留（不作为最终结论）：
  - 旧语句版本下的 benchmark、旧 Pareto/frontier、旧约束统计。
- 建议执行顺序：
  - 第 1 轮：小样本 smoke 重跑（确认量级变化）。
  - 第 2 轮：主搜索粗筛。
  - 第 3 轮：finalists 全量复测与稳定性回归。

## 7. 风险与缓解
- 风险：`Enc` 关系约束成本过高导致证明体积/时间显著上升。
  - 缓解：优先选用 ZK 友好加密组件或分层约束设计。
- 风险：旧接口兼容拖慢收口。
  - 缓解：final 与 compat 强隔离，明确弃用窗口。
- 风险：语句多口径实现导致一致性缺陷。
  - 缓解：单一语句派生函数 + 跨后端一致性测试强制门禁。

## 8. 完成定义（Definition of Done）
- 数学语句、代码约束、验证接口三者一致。
- final 验证语义为 Fischlin-Strict：接受等价于“见证存在且三关系成立”。
- 全量测试、基准、参数搜索在新语句下重跑完成并归档。
- 文档可直接支撑论文“协议定义/安全模型/实验结果”章节。
